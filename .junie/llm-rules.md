# LLM Project Rules

To maintain high code quality and consistency, all LLMs MUST follow these rules and NEVER modify the following files/directories unless explicitly requested for a foundational change:

## 1. Mandatory Workflow for LLMs

- **Follow README.md:** All rules defined in `README.md` are binding and MUST be followed strictly.
- **Run Checks:** After any code change, you MUST run the verification suite.
    - Run `npm run check:full` to execute the complete verification pipeline.
- **Verification Commands:**
    - `npm run i18n:generate`: Regenerates Paraglide-JS runtime (must run if translations or locales change).
    - `npm run check`: Runs Prettier (formatting + import sorting) and Oxlint (linting + auto-fixes).
    - `npx tsc --noEmit`: Runs TypeScript type checks.
    - `npm run check:full`: Combines all of the above into a single command for convenience.

## 2. Strictly Protected Paths (DO NOT MODIFY)

- `src/routeTree.gen.ts` (Auto-generated by TanStack Router)
- `src/paraglide/` (Auto-generated by Paraglide-JS)
- `ui/` (Shadcn/Base UI components - these are considered vendor/system components)

## 3. Core Guidelines

- **Follow Core Rules:** All LLMs MUST strictly adhere to the guidelines documented in [RULES](../RULES.md).
